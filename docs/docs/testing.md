# Testing

Тестирование JobCompass делится на автоматические тесты, smoke-тесты, regression-тесты и негативные кейсы.

Автоматический запуск на Pull Request, структура GitHub Actions jobs и учебный сценарий описаны на странице [CI/CD и практика автотестов](./ci-cd.md).

## Автоматические тесты backend

Запуск всех backend-тестов:

```bash
docker compose exec -e PYTHONPATH=/app api pytest
```

Запуск отдельного файла:

```bash
docker compose exec -e PYTHONPATH=/app api pytest tests/services/jobs/test_quality_pipeline.py
```

## Автоматические тесты frontend

```bash
cd jobcompass_ui
flutter test
```

Статический анализ:

```bash
cd jobcompass_ui
flutter analyze
```

## Smoke tests

Smoke tests - быстрый набор проверок, который отвечает на вопрос: "платформа в целом жива?"

Выполнять после крупных изменений, перед commit/push и перед публикацией GitHub Pages.

### Backend smoke

1. Запустить контейнеры:

```bash
docker compose up -d
```

2. Проверить health:

```bash
curl http://localhost:8000/health
```

Ожидаемо: HTTP 200 и сервис `JobCompass API`.

3. Проверить миграции:

```bash
docker compose exec api alembic current
```

Ожидаемо: база на актуальной миграции.

4. Запустить короткий набор backend-тестов:

```bash
docker compose exec -e PYTHONPATH=/app api pytest tests/schemas tests/services/jobs
```

### Frontend smoke

1. Запустить UI:

```bash
cd jobcompass_ui
flutter run -d chrome --web-port 5124
```

2. Открыть `http://localhost:5124`.
3. Проверить, что открывается экран входа.
4. Проверить, что название платформы отображается как `JobCompass`.
5. Зайти в основные разделы: Главная, Вакансии, Сохранённые, Аналитика, Профиль.
6. Для администратора проверить вкладку «Админ» справа от Профиля; для обычного пользователя вкладка отображаться не должна.

### Product smoke

1. Авторизоваться тестовым пользователем.
2. Заполнить профиль вручную без резюме и убедиться, что раздел Вакансии доступен.
3. Загрузить тестовое резюме и проверить, что ручные навыки не потерялись.
4. Открыть раздел Вакансии.
5. Убедиться, что у вакансий есть процент совпадения.
6. Сохранить вакансию.
7. Открыть сохранённую вакансию в разделе Сохранённые.
8. Оставить комментарий.
9. Открыть детали вакансии и проверить, что комментарий виден.
10. Ответить "Да" на вопрос об отклике.
11. Проверить появление вакансии в Аналитике.
12. Изменить статус отклика.
13. Архивировать вакансию.
14. Открыть архив и восстановить вакансию.

## Regression tests

Regression tests - проверки, которые подтверждают, что старые сценарии не сломались.

Выполнять перед крупным commit/push и после изменений в общих моделях, API или состоянии UI.

### Auth и профиль

- Вход с корректными данными.
- Ошибка входа с некорректным паролем.
- Создание и использование профиля без резюме.
- Сочетание ручных полей и загруженного резюме.
- Загрузка резюме.
- Замена резюме.
- Удаление резюме без удаления ручных полей.
- Проверка, что после перезапуска данные профиля загружаются из backend.

### Администрирование

- Обычный пользователь не видит вкладку «Админ» и получает HTTP 403 при прямом запросе к `/admin/*`.
- Администратор видит общее количество пользователей и список аккаунтов.
- В карточке пользователя администратор может выдать или отозвать бессрочный
  тариф Аналитики; новый статус сразу отображается в списке пользователей.
- На экране оплаты принимается произвольная сумма от 1 USDT; значения ниже 1,
  более 100 000 или с точностью больше двух знаков отклоняются до создания
  счёта.
- Открывается карточка любого пользователя и данные его профиля.
- Выдача и отзыв прав меняют статус аккаунта.
- Администратор не может отозвать права у самого себя.

### Вакансии

- Лента вакансий загружается.
- Фильтры по поиску и формату работы работают.
- Процент совпадения виден в карточке.
- Кнопка открытия деталей работает.
- Детали вакансии едины для Вакансий, Сохранённых и Аналитики.

### Сохранённые

- Сохранение вакансии.
- Удаление вакансии из сохранённых.
- Комментарий сохраняется.
- Комментарий синхронизируется с Вакансиями и Аналитикой.
- Процент совпадения отображается.

### Аналитика

- Отклик добавляет вакансию в Аналитику.
- Дубликаты вакансий не отображаются.
- Статус меняется.
- Статус `Тех. интервью` помещается в карточке.
- Архивирование убирает карточку из основного списка.
- Восстановление возвращает карточку из архива.
- Ручная Analytics сохраняется.
- In Progress считается автоматически.

### Локализация

- По умолчанию выбран русский язык.
- Переключение на английский меняет тексты.
- Ключевые экраны не показывают сырой ключ локализации вместо текста.

## Негативные кейсы

Негативные кейсы проверяют, что система корректно ведёт себя при ошибках.

### Backend

- Запрос без токена возвращает 401.
- Запрос с некорректным токеном возвращает 401.
- Запрос обычного пользователя к Admin API возвращает 403.
- Загрузка неподдерживаемого файла резюме не ломает backend.
- Повторное добавление той же вакансии не создаёт дубль.
- Внешний job provider недоступен: остальные источники продолжают работать.
- Некорректный ID заявки возвращает понятную ошибку.

### Frontend

- Backend выключен: UI показывает ошибку загрузки, а не пустой экран.
- Медленный backend: отображается loader.
- Некорректный ввод в Analytics: поле показывает ошибку.
- Пустой комментарий удаляет комментарий.
- Вакансия без stable identity не ломает карточку и детали.

### Database

- Миграция не применена: backend должен явно падать в логах, а не молча отдавать неправильные данные.
- Уникальный индекс не позволяет создать дубль job interaction.
- Удаление резюме не оставляет профиль в неконсистентном состоянии.
